# Tekton Guides

Website-owned, **cross-component** guides that intentionally span multiple
Tekton projects at once. This content does *not* live in any single component
repo and has no component release tag — it is authored directly in the website.

This is the IA need that the current Hugo + `sync.py` model cannot express:
the sync pipeline is strictly per-component, per-tag.
